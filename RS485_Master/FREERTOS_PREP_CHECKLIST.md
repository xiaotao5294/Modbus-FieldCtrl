# RS485_Master FreeRTOS 改造前准备清单

本文档用于把当前裸机工程稳定到“可迁移 FreeRTOS”的基线状态，避免迁移过程中出现优先级冲突、资源争抢、阻塞卡死和通信时序异常。

## 1. 当前已完成基线（代码已落地）

1. 中断优先级统一入口已建立：`App_PreflightInit()`。
2. 外设资源锁机制已建立：`App_PreflightLockResource()` / `App_PreflightUnlockResource()`。
3. USART 重复初始化防护已加入（USART1/USART2 一次性初始化保护）。
4. 控制台中断已瘦身：ISR 仅搬运字节到环形缓冲，解析在任务上下文执行。
5. 诊断与控制台发送路径已改为有限预算非阻塞发送，移除关键路径 `HAL_MAX_DELAY` 风险。
6. RS485 共享数据读写已加轻量锁抽象（后续可替换为 FreeRTOS Mutex）。

## 2. 改造前硬性验收项

1. 无阻塞验收
- 业务主路径禁止 `HAL_Delay`。
- 业务主路径禁止 `HAL_UART_Transmit(..., HAL_MAX_DELAY)`。
- 所有等待路径必须有超时退出条件。

2. 中断验收
- 中断处理函数只做最小工作（置标志、搬运字节、快速返回）。
- 复杂解析、格式化输出、策略决策必须放到任务上下文。
- 中断优先级策略统一由一个入口维护，不允许分散硬编码。

3. 资源验收
- 串口、DMA、TIM、ADC 资源占用要有唯一所有者。
- 重复初始化要被拒绝并可观测（计数或错误码）。
- 初始化顺序固定并可复现。

4. RS485/Modbus 验收
- 通信状态机持续非阻塞运行。
- 事务失败路径具备超时、重试、离线判定、恢复探测。
- 帧处理必须完整校验（地址、功能码、长度、CRC）后再执行控制动作。

## 3. 迁移前回归测试矩阵（建议逐条记录）

1. 正常通信
- 主机连续轮询 2 个从机 >= 30 分钟，无卡死。
- 成功计数持续增长，失败计数在可控范围内。

2. 故障注入
- 拔掉从机1：主机能进入离线状态并继续服务从机2。
- 恢复从机1：主机可自动探测恢复在线。
- 人为制造 CRC 错误：只记失败，不触发错误控制动作。

3. 并发压力
- 高频发送控制台命令（如 `THR=` 和 `SERVO=`）时，主通信不失步。
- 诊断打印开启时，轮询周期仍满足需求。

4. 边界输入
- 超范围阈值和角度输入被拒绝且系统不异常。
- 空命令、乱码、超长命令不会破坏命令状态机。

## 4. FreeRTOS 接入建议（下一阶段）

1. 任务划分建议
- `task_rs485_master`：协议状态机与事务调度（高优先级）。
- `task_console`：命令解析与回显（中优先级）。
- `task_diag`：周期诊断输出（低优先级）。

2. 同步原语映射建议
- `RS485_LockEnter/Exit` -> FreeRTOS Mutex。
- 控制台 RX 环形缓冲 -> Queue 或 StreamBuffer。
- 通信事件唤醒 -> Task Notification。

3. 时基建议
- 用 `vTaskDelayUntil()` 替代裸机 tick 轮询节拍控制。
- 所有超时统一采用 tick 计算，禁止裸常量散落。
