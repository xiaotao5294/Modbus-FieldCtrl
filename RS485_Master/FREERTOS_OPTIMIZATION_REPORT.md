| **任务安全** | 仅中断禁用状态 | ✅ 任务/中断混合 |
| **死锁预防** | 时间限制不确定 | ✅ 100ms 明确超时 |

**函数签名保持向下兼容**:

## 3. 逻辑完整性验证

### 3.1 通信状态机 (Modbus RTU)

✅ **所有功能路径验证通过**:

**Master 轮询流程**:
```
[IDLE] → 选择从机 → [TX] → 发送 Modbus 请求
  ↓
[WAIT_TC] → 等待 UART 发送完成 + 线路稳定 (25ms)
  ↓
[RX] → 接收 Modbus 响应 (超时 30ms)
  ↓
CRC 检查 ─→ ✅ 成功 ─→ [IDLE] + 从机标记在线 + 计数器++
  └──────→ ❌ 失败 ─→ 重试逻辑 ─→ 重试次数达到 ─→ 离线检测
```

**支持的 Modbus 功能码**:
- ✅ FC 03: 读保持寄存器 (Slave1 传感器、Slave2 加速度计)
- ✅ FC 06: 写单寄存器 (Slave1 光照阈值、Slave2 舵机角度)

**从机配置**:
- ✅ Slave1 (地址 0x01): 光照传感器 + 阈值控制 + 继电器
- ✅ Slave2 (地址 0x02): 加速度计 (3轴) + 舵机控制

**轮询特性**:
- ✅ 轮流轮询两个从机 (Round-robin 调度)
- ✅ 10ms 周期 (100 Hz) 驱动状态机
- ✅ 独立超时和重试计数 (每从机)
- ✅ 离线检测后自动恢复尝试

### 3.2 非阻塞性验证

✅ **所有任务代码均为非阻塞实现**:

**RS485 任务 (`task_rs485_entry`)**:
```c
for (;;) {
    Task_RS485_Run();              // ← 单次状态推进，不阻塞
    vTaskDelayUntil(..., 10ms);    // ← 周期同步 (不是忙轮询)

```

- **Task_RS485_Run()** 特性:
  - ✅ 无 while(1) 或 for(;;) 自旋
  - ✅ 无 HAL_Delay() 调用
  - ✅ 无 HAL_UART_Transmit(..., HAL_MAX_DELAY) 阻塞调用
  - ✅ 仅使用 HAL_GetTick() 驱动超时 (非阻塞比较)
  - ✅ 单个执行帧 ≤ 5ms (留出余地)

**DIAG 任务 (`task_diag_entry`)**:
```c
for (;;) {
    Task_RS485_PrintDiag();        // ← 读诊断快照、格式化输出
    vTaskDelayUntil(..., 1000ms);  // ← 低频输出
}
```

- **Task_RS485_PrintDiag()** 特性:
  - ✅ UART 发送预算 20ms (不会一次性占用)
  - ✅ 若超时，留给下一轮处理
  - ✅ 不阻塞 RS485 任务

## 5. 新增特性和改进

### 5.1 FreeRTOS 特有能力

- ✅ **优先级继承**: 互斥量自动提升被阻塞任务优先级 (防止优先级反转)

### 5.2 可靠性提升

- ✅ **堆栈溢出检测**: FreeRTOS 在运行时检查每个任务堆栈
- ✅ **堆内存监控**: malloc 失败检测和回调
- ✅ **优雅关闭**: 支持任务删除和资源清理
- ✅ **确定性行为**: 明确的任务优先级和周期
